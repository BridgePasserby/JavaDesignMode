中介(调停)者模式：
定义：Define an object than encapsulates how a set of objects interact.Mediator promotes loose soupling by keeping
objects from referring to each other explicitly,and it lets you vary their interaction independently.
用一个中介对象封装一系列的对象交互，中介者是个对象不续签显示地互相作用，从而使其耦合松散，而且可以独立地改变它们之间的交互。

角色：
Mediator抽象中介者角色
抽象中介者角色定义统一的接口，用于各同事角色之间的通信

Concrete Mediator 具体中介者角色
具体中介者角色通过协调各同事角色实现协作行为，因为它必须依赖于各同事角色

Colleague 同事角色
每一个同事角色都只是中介者角色，而且与其他的同事角色通信的时候，一点要通过中介者角色协作。

优点：
减少类间的依赖，把原有一对多的依赖变成了一对一的依赖，同事类只依赖中介者，减少了依赖，降低类间的耦合

缺点：
中介者会膨胀很大，而且逻辑复杂，原来N个对象直接的互相依赖关系转换为中介者和同事类的依赖关系，同事类越多，中介者的逻辑就
越复杂

适用场景：
1.N个对象之间产生了互相依赖的关系 N > 2
2.多个对象有依赖关系，但是依赖的行为尚不确定或者有发生改变的可能，在这种情况下一般建议采用中介者模式，降低变更引起的风
险扩散
3.MVC是一个典型的中介者模式

Demo:进销存模型，进、销、存之间互相依赖：
1.进货需要根据销量和库存数量决定
2.销售情况的变更会引起进货数量和库存数量的变化趋势，销售得好库存减少快且进货要增加
3.库存情况会限制进货数量或者打折销售

对比代理模式
1.代理模式是一对一，一个代理类只能代理一个对象。
    中介者是多对多，一个中介者可以有多个客户。

2.代理模式是只能单向访问，如果PB是A的代理，那么C可以通过PB访问A，但是A不能通过PB访问C
    中介者是双向访问，如果MB是A和C的中介，那么C可以通过MB访问A，A可以通过MB访问C
